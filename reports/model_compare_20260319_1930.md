# 모델 비교: LightGBM vs XGBoost vs CatBoost (Base / Tuned)

## 실험 조건

- LA=24h, DZ=1%, TTA(월간 재학습 720h), Fear&Greed 1일 shift 적용
- 6개 모델: LightGBM / XGBoost / CatBoost × Base / Tuned

### 하이퍼파라미터 비교

| 파라미터 | Base | Tuned |
|---|---|---|
| max_depth | 6 | 8 |
| learning_rate | 0.02 | 0.01 |
| num_leaves (LGBM) | 31 | 63 |
| feature_fraction / subsample | 0.7 | 0.6 |
| min_child_samples | 50 | 30 |
| lambda_l1 / reg_alpha | 0.1 | 0.05 |
| lambda_l2 / reg_lambda | 1.0 | 0.5 |
| early_stopping | 50 | 100 |
| max_boost_round | 1000 | 2000 |

## 분석

### 1. "튜닝 = 성능 향상"이 아니다

모든 모델에서 **Tuned 버전이 Base보다 나쁘거나 비슷**한 결과를 보임 (XGB 약세장 제외).

| 모델 | 2024~2025 Base→Tuned | 2025~2026 Base→Tuned |
|---|---|---|
| LGBM | +58.68% → +21.66% (**악화**) | -1.51% → -0.43% (소폭 개선) |
| XGB | +14.99% → +3.23% (**악화**) | +5.77% → +1.79% (**악화**) |
| CatBoost | -1.14% → +42.55% (**개선**) | -14.01% → -12.30% (소폭 개선) |

더 깊은 트리 + 낮은 LR = 더 복잡한 패턴 학습 → **과적합**. 노이즈를 학습하여 OOS 성능이 하락.
단, CatBoost는 내부 정규화가 강해서 tuned에서도 과적합이 상대적으로 적었음.

### 2. LGBM_base가 가장 안정적

- 강세장 **1위** (+58.68%, Sharpe 2.04)
- 약세장 4위 (-1.51%) — 하지만 B&H(-6.51%)보다 나음
- 학습 속도 **최고** (13.7s)
- 양쪽 기간 합산 수익률: +57.17%

### 3. XGB_base가 약세장에서 우위

- 약세장 **1위** (+5.77%, Sharpe 0.39)
- 강세장 4위 (+14.99%) — LGBM보다 보수적
- 양쪽 기간 합산 수익률: +20.76%
- MDD가 양쪽 모두 가장 작음 (-12.10%, -15.67%)

### 4. CatBoost는 불안정

- CatB_base: 양쪽 모두 최악 (-1.14%, -14.01%)
- CatB_tuned: 강세장 2위(+42.55%)이지만 약세장 최악(-12.30%)
- 학습 시간 과다 (513.8s)

### 5. 속도 효율

| 모델 | 평균 학습 시간 | 비고 |
|---|---|---|
| LGBM_base | ~19s | 최고 효율 |
| XGB_base | ~46s | 2.4x |
| CatB_base | ~50s | 2.6x |
| CatB_tuned | ~367s | **19x** — 비실용적 |

## 결론

1. **현재 LGBM_base 설정이 최선**. 튜닝이 오히려 과적합을 유발.
2. **XGB_base는 약세장 방어에 강점**이 있어, 앙상블 후보로 고려 가능.
3. CatBoost는 현재 파이프라인에서 비효율적 — 강세/약세 성능 편차가 큼.
4. 하이퍼파라미터 튜닝보다는 **피처 설계, 라벨링 방식, 의사결정 프레임워크(RL 등)** 변경이 더 근본적인 개선 방향.
5. GBDT 3종 모두 본질적으로 같은 regime 분류를 하고 있으므로, 모델을 바꾸는 것만으로는 한계가 있음.
